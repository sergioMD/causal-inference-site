Selection Bias

Why comparing treated and untreated groups often gives the wrong answer

In this section, you will learn to

  • Define selection bias as the part of the naive difference that reflects baseline differences between treated and untreated units rather than the treatment effect
  • Derive the decomposition \text{Naive} = \text{ATT} + \text{Selection bias} from the switching equation
  • State the mean-independence condition E[Y(0)\mid D=1] = E[Y(0)\mid D=0] under which the naive difference identifies the ATT
  • Sign the direction of bias from the selection pattern, explaining why positive selection overstates and negative selection understates the effect
  • Recognize that a bivariate OLS coefficient on D is identical to the naive difference, so regression repackages selection bias rather than removing it

The Problem with Naive Comparisons

Sweden’s 21 regioner (regions) are responsible for organizing healthcare within their territories. Suppose some regions adopted a digital vårdtjänst (digital health service) — a remote monitoring platform for heart failure patients that tracks daily weight, blood pressure, and symptoms. Of the 21 regions, 9 adopted the platform early; 12 did not. The obvious estimator — compare 30-day hospital readmission rates in adopting regions to those in non-adopting regions — is almost certainly biased.

The reason is selection. Regions that adopted the platform differ systematically from those that did not, and these differences are correlated with the outcome. Wealthier regions with strong IT infrastructure may have both better baseline outcomes and greater capacity to deploy digital tools. Conversely, regions with the worst readmission rates may have adopted precisely because they were desperate for improvement. In either case, the treated and untreated groups are not exchangeable, and the naive difference conflates the causal effect with pre-existing differences between groups.

This is selection bias: the component of the naive difference in means that reflects baseline differences between treated and untreated units, rather than the effect of treatment.

The picture below shows why this happens as a small causal diagram. A common cause — baseline severity / wealth — feeds both the decision to adopt (D) and the outcome (Y). That shared origin opens a non-causal “back-door” route (drawn in red) from D to Y alongside the genuine causal effect (green). The naive comparison cannot tell the two routes apart.

p2_drawDAG = function ({ nodes, edges, width = 440, height = 200, conditioned = [] }) {
  const pad = 48;
  const r = 24;
  const sx = v => pad + v * (width - 2 * pad);
  const sy = v => pad + v * (height - 2 * pad);
  const byId = new Map(nodes.map(d => [d.id, d]));
  const condSet = new Set(conditioned);

  const colorFor = k =>
    k === "causal" ? "#2e6f40" :
    (k === "biasing" || k === "backdoor-open") ? "#b3000b" : "#79766e";
  const markerFor = k =>
    k === "causal" ? "p2-arrow-green" :
    (k === "biasing" || k === "backdoor-open") ? "p2-arrow-red" : "p2-arrow-gray";

  const svg = d3.create("svg")
    .attr("width", width)
    .attr("height", height)
    .attr("viewBox", `0 0 ${width} ${height}`)
    .attr("font-family", "'Inter', sans-serif")
    .style("max-width", "100%")
    .style("height", "auto");

  const defs = svg.append("defs");
  for (const [id, col] of [
    ["p2-arrow-green", "#2e6f40"],
    ["p2-arrow-red", "#b3000b"],
    ["p2-arrow-gray", "#79766e"]
  ]) {
    defs.append("marker")
      .attr("id", id)
      .attr("viewBox", "0 -5 10 10")
      .attr("refX", 9).attr("refY", 0)
      .attr("markerWidth", 7).attr("markerHeight", 7)
      .attr("orient", "auto-start-reverse")
      .append("path").attr("d", "M0,-5L10,0L0,5").attr("fill", col);
  }

  const eg = svg.append("g");
  for (const e of edges) {
    const s = byId.get(e.source), t = byId.get(e.target);
    if (!s || !t) continue;
    const x1 = sx(s.x), y1 = sy(s.y), x2 = sx(t.x), y2 = sy(t.y);
    const dx = x2 - x1, dy = y2 - y1;
    const len = Math.hypot(dx, dy) || 1;
    const ux = dx / len, uy = dy / len;
    const xa = x1 + ux * r, ya = y1 + uy * r;
    const xb = x2 - ux * (r + 4), yb = y2 - uy * (r + 4);
    eg.append("line")
      .attr("x1", xa).attr("y1", ya)
      .attr("x2", xb).attr("y2", yb)
      .attr("stroke", colorFor(e.kind))
      .attr("stroke-width", e.kind === "blocked" ? 2 : 3)
      .attr("stroke-dasharray", e.kind === "blocked" ? "5,4" : null)
      .attr("marker-end", `url(#${markerFor(e.kind)})`);
  }

  const ng = svg.append("g");
  for (const n of nodes) {
    const cx = sx(n.x), cy = sy(n.y);
    const g = ng.append("g");
    if (condSet.has(n.id)) {
      g.append("rect")
        .attr("x", cx - r).attr("y", cy - r)
        .attr("width", 2 * r).attr("height", 2 * r)
        .attr("rx", 7).attr("ry", 7)
        .attr("fill", "#eef0ee").attr("stroke", "#1c1c1a").attr("stroke-width", 4);
    } else {
      g.append("circle")
        .attr("cx", cx).attr("cy", cy).attr("r", r)
        .attr("fill", "#eef0ee").attr("stroke", "#1c1c1a").attr("stroke-width", 2);
    }
    g.append("text")
      .attr("x", cx).attr("y", cy)
      .attr("text-anchor", "middle").attr("dominant-baseline", "central")
      .attr("font-size", 15).attr("font-weight", 700).attr("fill", "#1c1c1a")
      .text(n.label);
    if (n.caption) {
      g.append("text")
        .attr("x", cx).attr("y", cy + r + 15)
        .attr("text-anchor", "middle").attr("dominant-baseline", "central")
        .attr("font-size", 11).attr("font-weight", 600).attr("fill", "#79766e")
        .text(n.caption);
    }
  }

  return svg.node();
}

Decomposing the Naive Estimator

Let D_i \in \{0, 1\} indicate adoption. As a reminder of the potential-outcomes notation: Y_i(1) is unit i’s outcome if treated, Y_i(0) its outcome if untreated, and D_i the treatment indicator. The conditional mean E[Y_i(0)\mid D_i=1] then reads as “the readmission rate the adopting regions would have had if they had not adopted” — a counterfactual quantity. The naive comparison estimates:

\underbrace{E[Y_i | D_i = 1] - E[Y_i | D_i = 0]}_{\text{Naive difference}} = \underbrace{E[Y_i(1) - Y_i(0) | D_i = 1]}_{\text{ATT}} + \underbrace{E[Y_i(0) | D_i = 1] - E[Y_i(0) | D_i = 0]}_{\text{Selection bias}}

The naive difference equals the ATT only when the selection bias term is zero. That term measures the difference in untreated potential outcomes between the two groups — how different the adopting regions would have performed even absent the platform. When selection into treatment is correlated with the baseline outcome, this term is non-zero and the naive comparison is biased.

The naive difference in means identifies the ATT if and only if E[Y_i(0) | D_i = 1] = E[Y_i(0) | D_i = 0]. This is the mean independence condition. It requires that treatment assignment carries no information about baseline potential outcomes — a condition that is violated whenever selection is endogenous.

It only asks the average untreated outcome to match across the two groups — a weaker demand than full statistical independence (which would require their entire distributions to match). The naive difference needs nothing more than this single equality of means to be valid.

See It: Selection in Action

To make the statistics stable, the simulation expands the motivating example into a larger synthetic population of 300 units (think of them as regions pooled across many years) rather than just the 21 regioner. Use the slider to control the Selection Strength, which governs both how many units adopt the program and how strongly adoption depends on baseline severity.

  • At 0, treatment is assigned randomly. The treated and untreated groups are identical at baseline.
  • As you move to the right, “healthier” regions (higher Y(0)) are more likely to adopt.
  • As you move to the left, “sicker” regions are more likely to adopt.

This is an X-ray view, not a real analysis. The “Selection bias” and “Recovered ATT” readouts below are computed from the simulator’s hidden counterfactual — the Y(0) of treated regions, i.e. how they would have done without the platform. A real analyst never sees that column; it is exactly what the missing data make impossible to observe. The figure shows you the bias so you can learn to recognize it — not a recipe for removing it. (Removing it is what randomization and the later quasi-experimental designs are for.)

n = 300
trueATE = 4.0
seed = 42

function mulberry32(a) {
  return function() {
    a |= 0; a = a + 0x6D2B79F5 | 0;
    var t = Math.imul(a ^ a >>> 15, 1 | a);
    t = t + Math.imul(t ^ t >>> 7, 61 | t) ^ t;
    return ((t ^ t >>> 14) >>> 0) / 4294967296;
  }
}

function randn(rng) {
  let u1 = rng();
  let u2 = rng();
  return Math.sqrt(-2 * Math.log(u1)) * Math.cos(2 * Math.PI * u2);
}

data = {
  const rng = mulberry32(seed);
  const obs = [];
  for (let i = 0; i < n; i++) {
    const y0 = 50 + 10 * randn(rng);
    const y1 = y0 + trueATE + 2 * randn(rng);
    // Selection mechanism: probability of D depends on y0
    const propensity = 1 / (1 + Math.exp(-(selectionStrength * (y0 - 50) / 10)));
    const treated = rng() < propensity ? 1 : 0;
    const yObs = treated === 1 ? y1 : y0;
    obs.push({
      id: i, y0: y0, y1: y1, treated: treated,
      yObs: yObs, group: treated === 1 ? "Treated" : "Untreated"
    });
  }
  return obs;
}

// --- CALCULATIONS ---
stats = {
  const tr = data.filter(d => d.treated === 1);
  const un = data.filter(d => d.treated === 0);
  
  const mean_y_tr = d3.mean(tr, d => d.yObs) || 0;
  const mean_y_un = d3.mean(un, d => d.yObs) || 0;
  
  // Selection Bias = E[Y(0)|D=1] - E[Y(0)|D=0]
  const mean_y0_tr = d3.mean(tr, d => d.y0) || 50;
  const mean_y0_un = d3.mean(un, d => d.y0) || 50;
  
  const naive = mean_y_tr - mean_y_un;
  const bias = mean_y0_tr - mean_y0_un;
  const att = naive - bias; // Because Naive = ATT + Bias

  return {
    naive, bias, att, 
    trCount: tr.length, 
    unCount: un.length,
    mean_y_tr, mean_y_un
  }
}

How to Read This

  1. The Observed Outcome Gap: This is what the world looks like to a naive researcher. They see a gap between the red (treated) and slate (untreated) averages.
  2. The Bias Source: This looks only at the baseline (Y(0)) before treatment happened. If the red and slate lines aren’t perfectly aligned, your groups were different from the start.
  3. The Decomposition: This illustrates the exact identity Naive = ATT + Bias. The decomposed bar stacks the Bias segment from zero and the recovered ATT (\text{ATT} = \text{Naive} - \text{Bias}) on top of it, so its net top always lines up with the solid Naive bar — even when the bias is negative and the ATT segment pulls the total back up.

When selection is positive (healthy people join), the bias is positive and the Naive estimate makes the program look “better” than it really is. When selection is negative (sick people join), the bias is negative and can even make a successful program look like a failure!

Selection bias is not a statistical nuisance that can be fixed by collecting more data. It is a structural feature of the data-generating process. Increasing the sample size makes the naive estimate more precise, but it does not make it less biased. The bias converges to a non-zero constant. Eliminating selection bias requires a change in research design, not a larger dataset.

  const grid = [];
  const zs = d3.range(-50, 51).map(k => k * 0.1);      // z from -5 to 5
  const phi = z => Math.exp(-0.5 * z * z);             // unnormalized N(0,1) density
  for (let g = -3; g <= 3.0001; g += 0.1) {
    let wT = 0, wU = 0, sT = 0, sU = 0;
    for (const z of zs) {
      const y0 = 50 + 10 * z;
      const p = 1 / (1 + Math.exp(-(g * z)));          // matches sim: gamma*(y0-50)/10 = gamma*z
      const dens = phi(z);
      wT += p * dens;        sT += p * dens * y0;
      wU += (1 - p) * dens;  sU += (1 - p) * dens * y0;

  }
  return grid;
}

Direction of Bias

In practice, we rarely know the exact magnitude of the selection bias. But we can often reason about its direction. The sign of the selection bias depends on the sign of E[Y(0)|D=1] - E[Y(0)|D=0]: whether treated units have higher or lower baseline outcomes than untreated units.

Selection pattern \gamma Bias direction Naive estimate
Healthier regions adopt > 0 Positive Overstates effect
Sicker regions adopt < 0 Negative Understates effect
Adoption unrelated to baseline = 0 Zero Unbiased

This kind of qualitative reasoning is a core skill in applied work. Even when we cannot sign the bias precisely, knowing its likely direction disciplines how we interpret naive comparisons and motivates the identification strategies developed in subsequent modules.

Math Details: Formal Decomposition

The Selection Bias Decomposition

Start with the naive difference in means:

\Delta_{\text{naive}} = E[Y_i | D_i = 1] - E[Y_i | D_i = 0]

By the switching equation Y_i = D_i \cdot Y_i(1) + (1 - D_i) \cdot Y_i(0), this becomes:

\Delta_{\text{naive}} = E[Y_i(1) | D_i = 1] - E[Y_i(0) | D_i = 0]

Add and subtract E[Y_i(0) | D_i = 1]:

\Delta_{\text{naive}} = \underbrace{E[Y_i(1) | D_i = 1] - E[Y_i(0) | D_i = 1]}_{\text{ATT}} + \underbrace{E[Y_i(0) | D_i = 1] - E[Y_i(0) | D_i = 0]}_{\text{Selection Bias}}

Alternative decomposition targeting the ATE

We can also decompose the naive difference relative to the ATE by adding and subtracting both E[Y_i(0)|D_i=1] and E[Y_i(1)|D_i=0]:

\Delta_{\text{naive}} = \text{ATE} + \underbrace{E[Y_i(0)|D_i=1] - E[Y_i(0)|D_i=0]}_{\text{Selection bias}} + (1 - P(D=1)) \cdot \underbrace{(E[\tau_i|D_i=1] - E[\tau_i|D_i=0])}_{\text{Differential treatment effect bias}}

This makes clear that even if baseline outcomes are balanced (E[Y(0)|D=1] = E[Y(0)|D=0]), the naive estimator can still be biased for the ATE if treatment effects differ between groups.

y1 = y0 + true_ate + np.random.normal(0, 2, n)

        df.loc[df.treated == 0, "y_obs"].mean()

# Regression (biased — same problem, different packaging)
model = smf.ols("y_obs ~ treated", data=df).fit()

print(f"True ATE:             {true_ate:.2f}")
print(f"Naive diff-in-means:  {naive:.2f}")
print(f"OLS coefficient:      {model.params['treated']:.2f}")
print(f"Selection bias:       {naive - true_ate:.2f}")

Running a bivariate regression of Y on D does not solve the selection problem. The OLS coefficient on D is numerically identical to the naive difference in means. Regression provides a convenient framework for inference (standard errors, confidence intervals), but it does not change the estimand. The same selection bias that contaminates the simple difference also contaminates the regression coefficient.

Exercises

Exercise 1

A study compares test scores of students who voluntarily attended tutoring (D=1) with those who did not (D=0). The tutored students score 8 points higher on average. A skeptical colleague argues: “Students who seek tutoring are more motivated, so they would have scored higher anyway.” Formalize this argument using the selection bias decomposition. What is the sign of the bias? Does the naive estimate overstate or understate the effect of tutoring?

Show answer

The colleague’s argument, stated formally: students who seek tutoring have higher baseline test-taking ability, so E[Y(0)|D=1] > E[Y(0)|D=0]. This means the selection bias term is positive.

The decomposition gives us: 8 = \text{ATT} + \text{Bias}. Since bias > 0, it must be that \text{ATT} < 8. The naive estimate overstates the effect of tutoring. Part of the 8-point gap is real tutoring effect, but part is the pre-existing motivation difference.

For example, if the true ATT is 3 points, then the selection bias is 5 points — motivated students would have scored 5 points higher anyway, and tutoring added 3 more on top of that.

Exercise 2

Using the interactive simulation, set \gamma = -2 (negative selection). Observe the naive estimate, the true ATE, and the selection bias. Now set \gamma = +2. In which scenario is the naive estimate more misleading, and why? Can you find a \gamma where the naive estimate is negative even though the true ATE is positive?

Show answer

At \gamma = -2: “sicker” regions adopt the program. Selection bias is negative (the treated group has lower baseline outcomes). The naive estimate underestimates the true ATE and can even turn negative — making an effective program look harmful.

At \gamma = +2: “healthier” regions adopt. Selection bias is positive. The naive estimate inflates the true ATE, making the program look more effective than it is.

Both are misleading, but negative selection (\gamma = -2) is arguably more dangerous because it can reverse the sign of the estimated effect. By around \gamma \approx -1 the selection bias already exceeds the true effect in magnitude, so the naive estimate turns negative even though the true ATE is +4.0 (and it grows more negative as \gamma decreases further). The program genuinely helps, but the naive comparison makes it look like it hurts. A policymaker using this estimate would cancel an effective program.

Exercise 3

Prove that the naive difference in means equals the OLS coefficient from regressing Y_i on D_i (without additional covariates). Why does this imply that simple regression does not solve the selection bias problem?

Show answer

The OLS regression is Y_i = \alpha + \beta D_i + \varepsilon_i. The OLS formula for \hat{\beta} is:

\hat{\beta} = \frac{\text{Cov}(Y_i, D_i)}{\text{Var}(D_i)}

Since D_i is binary (0/1), we can show \text{Cov}(Y_i, D_i) = \bar{p}(1-\bar{p})(\bar{Y}_1 - \bar{Y}_0) and \text{Var}(D_i) = \bar{p}(1-\bar{p}), where \bar{p} is the fraction treated and \bar{Y}_1, \bar{Y}_0 are the group means. These cancel:

\hat{\beta} = \bar{Y}_1 - \bar{Y}_0 = \hat{\Delta}_{\text{naive}}

Alternatively: OLS minimizes \sum (Y_i - \alpha - \beta D_i)^2. The first-order conditions give \hat{\alpha} = \bar{Y}_0 and \hat{\alpha} + \hat{\beta} = \bar{Y}_1, so \hat{\beta} = \bar{Y}_1 - \bar{Y}_0.

Implication: Since the OLS coefficient is identical to the naive difference in means, it inherits the same selection bias. Running a regression does not fix the problem — it just repackages it. The regression gives you standard errors and p-values for the biased estimate, which can create a false sense of precision around the wrong answer.

Exercise 4

Consider the alternative decomposition targeting the ATE (from the Math Details section). Under what conditions does the “differential treatment effect bias” term equal zero? Give a substantive example where this term is likely non-zero.

Show answer

The alternative decomposition is:

\Delta_{\text{naive}} = \text{ATE} + \text{Selection bias} + (1 - P(D=1)) \cdot \underbrace{(E[\tau_i|D=1] - E[\tau_i|D=0])}_{\text{Differential treatment effect bias}}

The differential treatment effect bias term is zero when E[\tau_i|D=1] = E[\tau_i|D=0] — that is, when the average treatment effect is the same for those who select into treatment and those who don’t. This holds under:

  • Constant treatment effects: if \tau_i = \tau for all i, then both conditional expectations equal \tau.
  • Independence of gains from selection: if who benefits from treatment is unrelated to who gets treated (even if effects are heterogeneous).

Substantive example where it’s non-zero: Job training programs. Workers who enroll voluntarily may be those who know they need retraining (career changers, recently laid off from declining industries). These workers might benefit more from training than the average non-participant, who may already have marketable skills. Then E[\tau_i|D=1] > E[\tau_i|D=0] and the differential treatment effect bias is positive. This is “selection on gains” — people who expect to benefit more are more likely to participate.

Key takeaways

  • Selection bias is the term E[Y(0)\mid D=1] - E[Y(0)\mid D=0]: how different the treated and untreated groups would have been even with no treatment.
  • The naive difference identifies the ATT if and only if mean independence holds; endogenous selection breaks it.
  • The sign of the bias follows the selection pattern — healthier units adopting overstates the effect, sicker units adopting understates it (and can flip the sign).
  • Selection bias is structural, not a sampling-noise problem: more data makes the naive estimate more precise but no less biased; only a different research design removes it.
  • Running OLS of Y on D gives exactly the naive difference, so a regression coefficient inherits the same bias while adding a false air of precision.
