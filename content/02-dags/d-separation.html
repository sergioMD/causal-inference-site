d-Separation

The algorithmic backbone of causal reasoning with graphs

In this section, you will learn to

  • Apply the path-blocking rules to decide whether a path is open or blocked given a conditioning set, including the collider-descendant subtlety
  • Run the d-separation algorithm step by step to determine whether two variables are d-separated or d-connected by a set \mathbf{Z}
  • State and apply the backdoor criterion’s two conditions — no descendants of D, and every backdoor path blocked — to choose a valid adjustment set
  • Diagnose why conditioning on a mediator without first closing the backdoor yields an estimate with no clean causal interpretation
  • Recognize when no adjustment set identifies the effect (an unobserved confounder) and connect the backdoor criterion to conditional independence (ignorability)

From Intuition to Algorithm

The previous page introduced the three elemental structures — chains, forks, and colliders — and showed how conditioning opens or closes information flow in each case. But real-world DAGs have dozens of variables and hundreds of paths. You cannot eyeball a complex graph and decide which variables to condition on. You need an algorithm.

That algorithm is d-separation (the “d” stands for directional). Given a DAG, d-separation tells you whether two variables are independent, conditional on a set of other variables — without looking at any data. It turns causal questions into graph-tracing exercises.

This page teaches you d-separation from the ground up, then applies it to the central question of causal inference: which variables must I condition on to identify a causal effect? The answer is the backdoor criterion.

Blocking and Opening Paths

Recall from the basics page: a path between two variables is any route connecting them, following edges in either direction. Information can flow along a path unless something blocks it.

The rules for what blocks a path depend entirely on the three elemental structures:

Path-blocking rules — A path between X and Y is blocked by a conditioning set \mathbf{Z} if and only if the path contains at least one of the following:

  1. A chain \dots \rightarrow M \rightarrow \dots or a fork \dots \leftarrow C \rightarrow \dots where the middle variable (M or C) is in \mathbf{Z} (i.e., you are conditioning on it). Conditioning closes these structures.

  2. A collider \dots \rightarrow L \leftarrow \dots where L is not in \mathbf{Z} and no descendant of L is in \mathbf{Z}. Colliders are blocked by default — they only open when you condition on them (or on their descendants).

How to read this: Walk along the path one node at a time. At each node, check: is this node a chain/fork middle that I’m conditioning on? If yes, the path is blocked — stop. Is this node a collider? If I’m not conditioning on it (or any of its descendants), the path is blocked — stop. If you reach the end without being blocked, the path is open.

The descendant subtlety: Conditioning on a descendant of a collider also partially opens the collider path. For example, if A \rightarrow L \leftarrow B and L \rightarrow M, conditioning on M (a child of the collider L) also opens the path between A and B. This is because observing M gives partial information about L, which is enough to induce an association. Always check whether your conditioning set includes any descendants of colliders on the path.

d-Separation: The Formal Definition

Two variables X and Y are d-separated by a set \mathbf{Z} in a DAG \mathcal{G} if every path between X and Y is blocked by \mathbf{Z}.

X \perp_{\mathcal{G}} Y \mid \mathbf{Z} \quad \Leftrightarrow \quad \text{every path between } X \text{ and } Y \text{ is blocked by } \mathbf{Z}

How to read this: “X is d-separated from Y given \mathbf{Z} in graph \mathcal{G}” means that, under the causal assumptions encoded in the DAG, X and Y should be statistically independent once you condition on \mathbf{Z}. The subscript \mathcal{G} reminds us that d-separation is a property of the graph, not the data.

If X and Y are not d-separated by \mathbf{Z}, they are d-connected: at least one open path remains, and we should expect an association in the data.

The d-Separation Algorithm (Step by Step)

To determine whether X \perp_\mathcal{G} Y \mid \mathbf{Z}:

  1. List all paths between X and Y (ignoring arrow directions, just trace routes through the graph).
  2. For each path, walk along it and check each intermediate node:
    • If the node is a non-collider (chain middle or fork center) and it is in \mathbf{Z} → the path is blocked at this node. Stop checking this path.
    • If the node is a collider and neither it nor any of its descendants is in \mathbf{Z} → the path is blocked at this collider. Stop checking this path.
    • Otherwise, the node does not block the path. Continue to the next node.
  3. If a path has no blocking nodes, it is open.
  4. If all paths are blocked, X and Y are d-separated given \mathbf{Z}. If any path is open, they are d-connected.

A Worked Example: Swedish Maternity Care

Consider a study of how a prenatal care intervention (D) affects birth weight (Y) in a Swedish hospital. A researcher proposes the following DAG:

D \leftarrow \text{SES} \rightarrow Y D \rightarrow \text{Prenatal visits} \rightarrow Y \text{SES} \rightarrow \text{Prenatal visits}

The variables:

  • D = enrollment in the specialized prenatal program
  • Y = birth weight
  • SES = socioeconomic status of the mother
  • Prenatal visits = number of prenatal care visits during pregnancy

Goal: Is D d-separated from Y given \emptyset (the empty set, i.e., no conditioning)?

Step 1 — List all paths from D to Y:

  1. D \rightarrow \text{Prenatal visits} \rightarrow Y (causal, through the mediator)
  2. D \leftarrow \text{SES} \rightarrow Y (backdoor, through the confounder)
  3. D \leftarrow \text{SES} \rightarrow \text{Prenatal visits} \rightarrow Y (another backdoor, longer route)

Step 2 — Check each path for blocking (conditioning on \mathbf{Z} = \emptyset):

  1. D \rightarrow \text{Prenatal visits} \rightarrow Y: Prenatal visits is a chain middle, not in \mathbf{Z}. Path is open.
  2. D \leftarrow \text{SES} \rightarrow Y: SES is a fork center, not in \mathbf{Z}. Path is open.
  3. D \leftarrow \text{SES} \rightarrow \text{Prenatal visits} \rightarrow Y: SES is a fork center, not in \mathbf{Z}; Prenatal visits is a chain middle, not in \mathbf{Z}. Path is open.

Conclusion: D and Y are d-connected given \emptyset. The association between D and Y includes both causal and non-causal components.

Now condition on SES (\mathbf{Z} = \{\text{SES}\}):

  1. D \rightarrow \text{Prenatal visits} \rightarrow Y: No node is conditioned on. Path remains open ✓ (this is the causal path — we want it open).
  2. D \leftarrow \text{SES} \rightarrow Y: SES is in \mathbf{Z}, and it’s a fork center. Path is blocked ✓.
  3. D \leftarrow \text{SES} \rightarrow \text{Prenatal visits} \rightarrow Y: SES is in \mathbf{Z}. Path is blocked at SES ✓.

After conditioning on SES, only the causal path remains open. The association between D and Y given SES is the causal effect.

  const condM = condSet.includes("M (Nurse ratio)");
  const condC = condSet.includes("C (Reputation)");

  const d = dagData;
  const n = d.length;

  // Project `vec` onto the orthogonal complement of a single (mean-centered) basis vector.
  function sweepOut(vec, basis) {
    const mb = d3.mean(basis);
    const vb = d3.sum(basis, b => (b - mb) ** 2);
    if (vb === 0) return vec.slice();
    const cov = d3.sum(basis.map((b, i) => (b - mb) * vec[i]));
    const beta = cov / vb;
    return vec.map((r, i) => r - beta * (basis[i] - mb));
  }

  // Residualize via OLS. For >=2 correlated controls, sequential univariate
  // residualization is NOT equivalent to multivariate OLS, so we first build a
  // mutually orthogonal basis from the controls (Gram-Schmidt) and then sweep the
  // outcome against that basis. This yields the true partial regression coefficient.
  function residualize(outcome, controls) {
    if (controls.length === 0) return outcome.slice();
    const basis = [];
    for (let ctrl of controls) {
      for (const b of basis) ctrl = sweepOut(ctrl, b);
      basis.push(ctrl);
    }
    let resid = outcome.slice();
    for (const b of basis) resid = sweepOut(resid, b);
    return resid;
  }

  const controls = [];
  if (condU) controls.push(d.map(v => v.U));
  if (condM) controls.push(d.map(v => v.M));
  if (condC) controls.push(d.map(v => v.C));

  const residD = residualize(d.map(v => v.D), controls);
  const residY = residualize(d.map(v => v.Y), controls);

  const mrD = d3.mean(residD);
  const mrY = d3.mean(residY);

The Backdoor Criterion

d-Separation gives us the general tool. The backdoor criterion is the most important special case: it tells us which variables to condition on to identify the total causal effect of D on Y.

The Backdoor Criterion (Pearl, 1993)

A set of variables \mathbf{Z} satisfies the backdoor criterion relative to the treatment D and outcome Y if:

  1. No variable in \mathbf{Z} is a descendant of D (don’t condition on anything caused by the treatment).
  2. \mathbf{Z} blocks every backdoor path from D to Y (every non-causal path that starts with an arrow into D).

If \mathbf{Z} satisfies the backdoor criterion, then the causal effect of D on Y is identified by the adjustment formula:

P(Y \mid do(D = d)) = \sum_{\mathbf{z}} P(Y \mid D = d, \mathbf{Z} = \mathbf{z}) \, P(\mathbf{Z} = \mathbf{z})

How to read this formula: The left side, P(Y \mid do(D = d)), is the distribution of outcomes if we intervened and set everyone’s treatment to d. The do(\cdot) operator distinguishes intervention (physically setting a value) from observation (passively recording it). The right side says: compute the conditional association between D and Y within each stratum of \mathbf{Z}, then average across strata weighted by the population distribution of \mathbf{Z}.

In practice (for continuous variables and linear models), this reduces to: regress Y on D and \mathbf{Z}, and the coefficient on D estimates the causal effect.

Why the Two Conditions?

Condition 1 (no descendants of D) prevents you from conditioning on mediators or colliders downstream of treatment. Conditioning on a mediator blocks part of the causal effect. Conditioning on a collider that is a descendant of D can open new spurious paths.

Condition 2 (block all backdoor paths) ensures that every non-causal association between D and Y is removed. A backdoor path starts with an arrow into D — these paths carry confounding, not causation.

The backdoor criterion has a remarkable implication: you don’t need to observe every confounder. You need to observe enough variables to block every backdoor path. Sometimes a single variable blocks multiple paths. Sometimes you can block a path by conditioning on a variable that isn’t itself a confounder but sits on a backdoor path. The DAG shows you the minimum you need — and equally, when no sufficient adjustment set exists because a confounder is unobserved and unreachable.

Multiple Valid Adjustment Sets

An important insight: the backdoor criterion often admits multiple valid adjustment sets. The maternity care example above is deliberately minimal — it has just one confounder (SES), so \{\text{SES}\} is the only valid adjustment set. It is worth seeing why the obvious alternatives fail:

  • \{\text{Prenatal visits}\} alone does not satisfy the criterion: it fails to block the backdoor path D \leftarrow \text{SES} \rightarrow Y.
  • \{\text{SES}, \text{Prenatal visits}\} is a trap. It does block every backdoor path, but Prenatal visits is a child of D (D \rightarrow \text{Prenatal visits}), so it is a descendant of the treatment and violates Condition 1 of the backdoor criterion. Worse, Prenatal visits is a mediator on the causal path D \rightarrow \text{Prenatal visits} \rightarrow Y; conditioning on it blocks part of the very effect we are trying to estimate. The resulting coefficient is neither the total effect nor a clean direct effect.

The lesson: blocking all backdoor paths (Condition 2) is necessary but not sufficient. A valid set must also contain no descendants of D (Condition 1).

To see multiple genuinely valid sets, you need a graph with redundant blocking points — as in Exercise 1 below, where the backdoor D \leftarrow \text{Clinic size} \rightarrow \text{Staff training} \rightarrow Y can be blocked at either node, giving two minimal valid sets: \{\text{Clinic size}\} and \{\text{Staff training}\}.

When multiple valid sets do exist, how should you choose?

  1. Prefer smaller sets (fewer variables) for simplicity and statistical efficiency.
  2. Avoid mediators if you want the total effect (conditioning on a mediator gives you a direct effect, which may not be what you want).
  3. Prefer well-measured variables — conditioning on a poorly measured confounder only partially blocks the path (residual confounding).
  4. Stay upstream of treatment — descendants of D are risky even if they technically satisfy the criterion.

When Identification Fails

Sometimes no adjustment set satisfies the backdoor criterion. This happens when a backdoor path runs through a variable that is unobserved and cannot be blocked by any observed variable.

Consider a study of whether a new physiotherapy method (D) reduces chronic pain (Y) at a Swedish rehabiliteringsklinik:

D \leftarrow U \rightarrow Y

where U is the patient’s underlying “motivation and pain tolerance” — unobserved and unmeasured. No other variable lies on this path. Nothing we can condition on will block it.

The figure makes the obstruction visible: U is drawn as a dashed, hollow node to mark it as unobserved, and the back-door fork it creates is red and permanently open — there is no observed node sitting on the path to box.

When the backdoor criterion fails, you need a different identification strategy: instrumental variables (Module 3), regression discontinuity (Module 4), difference-in-differences (Module 6), or other methods covered later. The DAG doesn’t just tell you when conditioning works — it tells you when it doesn’t, which is equally valuable.

The Connection to Potential Outcomes

How does d-separation connect to the potential outcomes framework from Module 1? The bridge is the conditional independence assumption (also called “selection on observables,” “ignorability,” or “unconfoundedness”):

(Y(0), Y(1)) \perp\!\!\!\perp D \mid \mathbf{Z}

How to read this: “The potential outcomes Y(0) and Y(1) are jointly independent of the treatment D, conditional on \mathbf{Z}.” In words: after conditioning on \mathbf{Z}, the treatment assignment is as good as random — treated and untreated groups are comparable within each stratum of \mathbf{Z}.

The backdoor criterion provides the graphical condition under which this statistical assumption holds. If \mathbf{Z} satisfies the backdoor criterion in your DAG, and the DAG is correct, then the conditional independence assumption is satisfied.

Math Details: Faithfulness, Testability, and the Adjustment Formula

The Faithfulness Assumption

The Markov condition (from the basics page) says: d-separation in the graph implies independence in the data.

X \perp_{\mathcal{G}} Y \mid \mathbf{Z} \implies X \perp\!\!\!\perp_P Y \mid \mathbf{Z}

How to read this: “If X and Y are d-separated by \mathbf{Z} in graph \mathcal{G}, then X and Y are statistically independent given \mathbf{Z} in the data distribution P.” The arrow “\implies” means “implies” — the left side guarantees the right side.

Faithfulness adds the reverse direction:

X \not\perp_{\mathcal{G}} Y \mid \mathbf{Z} \implies X \not\perp\!\!\!\perp_P Y \mid \mathbf{Z}

How to read this: “If X and Y are d-connected given \mathbf{Z}, then they are associated in the data.” Faithfulness rules out knife-edge cases where two confounding paths accidentally cancel each other out. Such cancellations have probability zero in continuous distributions but can occur in carefully engineered examples.

Why Faithfulness Matters for Testability

Together, the Markov condition and faithfulness imply that a DAG generates a specific set of conditional independence relations. You can test some of these in data. If the data show an independence that the DAG does not predict (or vice versa), the DAG may be wrong. This gives DAGs a degree of empirical falsifiability, even though you cannot confirm them from data alone.

The Adjustment Formula (Derivation Sketch)

Given that \mathbf{Z} satisfies the backdoor criterion:

P(Y \mid do(D = d)) = \sum_{\mathbf{z}} P(Y \mid D = d, \mathbf{Z} = \mathbf{z}) \, P(\mathbf{Z} = \mathbf{z})

The intuition: the do(\cdot) operator is defined by the truncated factorization. In the observational DAG, the joint distribution factors as:

P(V) = \prod_{j} P(V_j \mid \text{Parents}(V_j))

When we set D = d by intervention, we delete the factor P(D \mid \text{Parents}(D)) (because D is no longer determined by its parents — it is set externally) and replace it with a point mass:

P(V \mid do(D = d)) = \prod_{j \ne D} P(V_j \mid \text{Parents}(V_j)) \Big|_{D=d}

How to read this: “The post-intervention distribution equals the original factorization with the treatment’s equation removed and D fixed at d.” This “graph surgery” removes all arrows into D, which is equivalent to deleting all backdoor paths.

The backdoor criterion ensures that conditioning on \mathbf{Z} achieves the same result statistically, without requiring physical intervention. The proof uses the three rules of Pearl’s do-calculus (1995).

Exercise 1

Consider the following DAG for a study of how electronic health records (D) affect diagnostic accuracy (Y) in Swedish vårdcentraler:

D \leftarrow \text{Clinic size} \rightarrow \text{Staff training} \rightarrow Y D \rightarrow Y

List all paths from D to Y. For each path, determine whether it is causal or non-causal. Then find all minimal valid adjustment sets using the backdoor criterion.

Show answer

Paths from D to Y:

  1. D \rightarrow Y — causal (direct effect). No intermediate nodes to block.
  2. D \leftarrow \text{Clinic size} \rightarrow \text{Staff training} \rightarrow Y — backdoor (non-causal). Starts with an arrow into D.

Applying the backdoor criterion:

We need to block path 2 without conditioning on descendants of D.

  • \mathbf{Z} = \{\text{Clinic size}\} — blocks path 2 at the fork center. Valid ✓
  • \mathbf{Z} = \{\text{Staff training}\} — blocks path 2 at the chain middle. Valid ✓ (Staff training is not a descendant of D.)
  • \mathbf{Z} = \{\text{Clinic size}, \text{Staff training}\} — blocks path 2 redundantly. Valid but not minimal.
  • \mathbf{Z} = \emptyset — path 2 is open. Invalid ✗

The two minimal valid sets are \{\text{Clinic size}\} and \{\text{Staff training}\}.

Exercise 2

In the d-Separation Explorer above, what happens when you condition on only M (the mediator)? The backdoor path is still open, but the estimate changes. Explain what the estimate represents and why it has no clean causal interpretation.

Show answer

When you condition on M (the nurse-to-patient ratio), two things happen simultaneously:

  1. The causal path D \rightarrow M \rightarrow Y is blocked: Conditioning on the mediator removes the indirect causal effect (0.6 \times 0.4 = 0.24). Only the direct effect (D \rightarrow Y = 0.30) survives on the causal side.

  2. The backdoor path D \leftarrow U \rightarrow C \rightarrow Y remains open: Nothing blocks the confounding flow through U and C.

So the estimate captures: the direct causal effect (0.30) plus the confounding bias from the open backdoor. These combine into a number that is neither the total causal effect (0.54) nor the controlled direct effect (0.30) nor anything with a clean interpretation.

This illustrates a critical lesson: conditioning on a mediator without first blocking the backdoor produces an uninterpretable estimate. The first condition of the backdoor criterion (“no descendants of D”) exists precisely to prevent this.

Exercise 3

A researcher at a Swedish university hospital proposes the following DAG for studying whether a new surgical technique (D) reduces complications (Y):

D \leftarrow \text{Surgeon skill} \rightarrow Y D \rightarrow \text{Operating time} \leftarrow \text{Patient BMI} \rightarrow Y

  1. Is “Operating time” a collider? On which path?
  2. What happens if the researcher conditions on Operating time?
  3. Find the minimal valid adjustment set.
Show answer

(a) Yes, Operating time is a collider on the path D \rightarrow \text{Op. time} \leftarrow \text{BMI} \rightarrow Y. Both D and BMI have arrows pointing into Operating time.

(b) Conditioning on Operating time opens the collider path: D \rightarrow \text{Op. time} \leftarrow \text{BMI} \rightarrow Y. This creates a spurious association between D and BMI (through the opened collider), which then flows to Y. The researcher introduces bias that wasn’t there before.

(c) List the paths from D to Y:

  1. D \rightarrow Y (if a direct arrow exists) — causal.
  2. D \leftarrow \text{Surgeon skill} \rightarrow Y — backdoor, open.
  3. D \rightarrow \text{Op. time} \leftarrow \text{BMI} \rightarrow Y — contains a collider at Op. time. This path is blocked by default (the collider is not conditioned on).

Only path 2 is a problem. The minimal valid set is \mathbf{Z} = \{\text{Surgeon skill}\}. This blocks the only open backdoor. The collider path is already blocked. Do not condition on Operating time.

Exercise 4

A common piece of advice in epidemiology is: “control for everything that is correlated with both treatment and outcome.” Using what you know about d-separation and the three elemental structures, explain why this advice is wrong. Give a concrete healthcare example where following this advice introduces bias.

Show answer

The advice “control for everything correlated with both treatment and outcome” is wrong because correlation does not distinguish confounders from colliders.

A confounder (fork: D \leftarrow C \rightarrow Y) is correlated with both D and Y and should be controlled for. A collider (common effect: D \rightarrow L \leftarrow Y) is also correlated with both D and Y (because they are its causes!) but should not be controlled for. Conditioning on a collider creates bias.

Concrete example: A Swedish hospital studies whether a new medication (D) reduces 30-day readmission (Y). Both the medication and the severity of the initial illness contribute to whether the patient is admitted to the ICU (L):

D \rightarrow L \leftarrow Y

L (ICU admission) is correlated with both D and Y. Following the bad advice, the researcher controls for ICU admission. This is a collider: conditioning on it creates a spurious association. Among ICU patients, those on the new medication appear to have worse readmission rates — not because the drug is harmful, but because non-medication factors (like severity) must have been especially high to land them in the ICU.

The correct approach: draw a DAG, check which variables are confounders vs. colliders vs. mediators, and use the backdoor criterion. The presence of correlation with treatment and outcome is necessary but not sufficient to justify adjustment.

Key takeaways

  • d-Separation turns causal questions into graph-tracing: X and Y are d-separated by \mathbf{Z} only if every path between them is blocked — one open path makes them d-connected.
  • A non-collider (chain middle or fork center) blocks a path when it is in \mathbf{Z}; a collider blocks by default and opens only when it — or any descendant — is in \mathbf{Z}.
  • The backdoor criterion identifies the total effect when \mathbf{Z} contains no descendant of D and blocks every backdoor path; blocking the backdoor alone is necessary but not sufficient.
  • You need not observe every confounder — only enough variables to block every backdoor path, and the same path can often be blocked at more than one node.
  • When a backdoor path runs through an unobserved variable no observed variable can block, the effect is not identified, and you need IV, RD, or DiD instead.
