Good and Bad Controls

When conditioning helps, and when it hurts

In this section, you will learn to

  • Classify a candidate control by its causal role — confounder, proxy, mediator, collider, collider descendant, or precision variable — rather than by its correlation with treatment and outcome
  • Distinguish good controls (block a backdoor) from bad controls that cause overcontrol bias (conditioning on a mediator) or collider/selection bias
  • Explain why “control for everything pre-treatment” fails, using M-bias (the butterfly structure) where a baseline collider opens a path
  • Separate bias from sampling noise by reading the center versus the spread of an estimator’s sampling distribution
  • Apply a step-by-step decision framework for any candidate control and recognize the Table 2 fallacy — that one regression answers only one causal question

The Stakes of Getting It Wrong

A Swedish researcher wants to know whether a new antihypertensive drug (D) reduces stroke risk (Y). She has data on thousands of patients from the Swedish National Patient Register (Patientregistret), with dozens of covariates. Her plan: “I’ll just control for everything I have.”

This is one of the most common — and most dangerous — strategies in applied research. Whether a variable should be in your regression depends on its causal role, not on its predictive power or its correlation with treatment and outcome. The previous pages gave you the tools: DAGs and d-separation. This page applies those tools to the six most common adjustment patterns, organized as good controls (variables that reduce bias) and bad controls (variables that introduce or amplify bias). Each pattern is paired with a small DAG figure that color-codes which paths are open and which are blocked, and three live simulations let you experience the consequences: a redesigned Control Laboratory that separates bias from sampling noise, an “Explaining-Away” demo that manufactures a correlation from thin air by conditioning on a collider, and a butterfly / M-bias demo showing how even a pre-treatment control can open a biasing path.

The core principle: A control variable is “good” if conditioning on it blocks a non-causal path (reducing bias) without opening other non-causal paths or blocking causal paths. A control variable is “bad” if conditioning on it opens a non-causal path, blocks a causal path you wanted open, or does both. The DAG tells you which is which.

Good Controls

1. The Classic Confounder

D \leftarrow C \rightarrow Y

C is a common cause of both treatment and outcome. This is the fork structure from the basics page. The backdoor path D \leftarrow C \rightarrow Y is open, creating a spurious association.

Action: Condition on C. This blocks the backdoor path.

Healthcare example: Studying the effect of physical therapy (D) on recovery time (Y) after hip replacement in a Swedish hospital. Patients’ pre-operative fitness (C) affects both whether they receive intensive therapy (fitter patients are more likely to be offered it) and how quickly they recover. Conditioning on pre-operative fitness removes this confounding.

The Confounder Pattern

  • Structure: D \leftarrow C \rightarrow Y (fork)
  • Without conditioning: D and Y are spuriously associated through C
  • With conditioning on C: Backdoor path blocked, only causal association remains
  • Rule: Always condition on common causes of treatment and outcome

The figure below renders this structure. Throughout this page, a small DAG accompanies each pattern: green arrows are open causal paths, red arrows are open non-causal (biasing) paths, gray dashed arrows are blocked, and a boxed node is one you are conditioning on.

// ─── Shared DAG component for all control-pattern figures on this page ───
// Pure function: builds an SVG with d3.create and returns the DOM node.
// nodes: [{id, x, y, label}] with x,y in normalized [0,1] coordinates.
// edges: [{source, target, status}] status ∈ "open-causal"|"open-noncausal"|"blocked".
// conditioned: [id, ...] — those nodes render as boxed squares, others as circles.
gdag_drawDAG = function({nodes, edges, width = 440, height = 230, conditioned = []}) {
  const pad = 38;                 // keep nodes off the edges
  const r = 20;                   // node radius
  const sx = x => pad + x * (width - 2 * pad);
  const sy = y => pad + y * (height - 2 * pad);

  const colorFor = status =>
    status === "open-causal" ? "#2e6f40"
    : status === "open-noncausal" ? "#b3000b"
    : "#79766e";

  const byId = new Map(nodes.map(n => [n.id, n]));

  const svg = d3.create("svg")
    .attr("width", width)
    .attr("height", height)
    .attr("viewBox", `0 0 ${width} ${height}`)
    .attr("font-family", "'Inter', sans-serif")
    .style("max-width", "100%")
    .style("height", "auto");

  // One arrowhead marker per edge color (defined once).
  const defs = svg.append("defs");
  for (const [name, col] of [["causal", "#2e6f40"], ["noncausal", "#b3000b"], ["blocked", "#79766e"]]) {
    defs.append("marker")
      .attr("id", `gdag-arrow-${name}`)
      .attr("viewBox", "0 -5 10 10")
      .attr("refX", 9)
      .attr("refY", 0)
      .attr("markerWidth", 7)
      .attr("markerHeight", 7)
      .attr("orient", "auto")
      .append("path")
      .attr("d", "M0,-5L10,0L0,5")
      .attr("fill", col);
  }

  // ─ Edges first (drawn under the nodes) ─
  const gEdges = svg.append("g");
  for (const e of edges) {
    const s = byId.get(e.source), t = byId.get(e.target);
    if (!s || !t) continue;
    const x1 = sx(s.x), y1 = sy(s.y), x2 = sx(t.x), y2 = sy(t.y);
    const dx = x2 - x1, dy = y2 - y1;
    const len = Math.sqrt(dx * dx + dy * dy) || 1;
    const ux = dx / len, uy = dy / len;
    // Trim the line to the node borders so the arrowhead sits on the rim.
    const fx = x1 + ux * r, fy = y1 + uy * r;
    const tx = x2 - ux * (r + 4), ty = y2 - uy * (r + 4);
    const markerName = e.status === "open-causal" ? "causal"
      : e.status === "open-noncausal" ? "noncausal" : "blocked";
    if (e.curve) {
      // Bow the edge into an arc so it reads as distinct from any node/edge
      // running through the straight corridor between source and target. The
      // sign of e.curve (perpendicular offset, px) sets the bow direction.
      const mx = (fx + tx) / 2, my = (fy + ty) / 2;
      const px = -uy, py = ux;            // unit normal to the edge
      const cxp = mx + px * e.curve, cyp = my + py * e.curve;
      gEdges.append("path")
        .attr("d", `M${fx},${fy} Q${cxp},${cyp} ${tx},${ty}`)
        .attr("fill", "none")
        .attr("stroke", colorFor(e.status))
        .attr("stroke-width", e.status === "blocked" ? 2 : 2.6)
        .attr("stroke-dasharray", e.status === "blocked" ? "5,4" : null)
        .attr("marker-end", `url(#gdag-arrow-${markerName})`);
    } else {
      gEdges.append("line")
        .attr("x1", fx).attr("y1", fy)
        .attr("x2", tx).attr("y2", ty)
        .attr("stroke", colorFor(e.status))
        .attr("stroke-width", e.status === "blocked" ? 2 : 2.6)
        .attr("stroke-dasharray", e.status === "blocked" ? "5,4" : null)
        .attr("marker-end", `url(#gdag-arrow-${markerName})`);
    }
  }

  // ─ Nodes on top ─
  const gNodes = svg.append("g");
  for (const n of nodes) {
    const cx = sx(n.x), cy = sy(n.y);
    const isCond = conditioned.includes(n.id);
    const g = gNodes.append("g");
    if (isCond) {
      g.append("rect")
        .attr("x", cx - r).attr("y", cy - r)
        .attr("width", 2 * r).attr("height", 2 * r)
        .attr("rx", 5)
        .attr("fill", "#f6f5f1")
        .attr("stroke", "#1c1c1a")
        .attr("stroke-width", 4);
    } else {
      g.append("circle")
        .attr("cx", cx).attr("cy", cy).attr("r", r)
        .attr("fill", "#f6f5f1")
        .attr("stroke", "#1c1c1a")
        .attr("stroke-width", 2);
    }
    g.append("text")
      .attr("x", cx).attr("y", cy)
      .attr("text-anchor", "middle")
      .attr("dominant-baseline", "central")
      .attr("font-size", 14)
      .attr("font-weight", 700)
      .attr("fill", "#1c1c1a")
      .text(n.label);
  }

  return svg.node();
}
gdag_drawDAG({
  width: Math.min(420, width), height: 170,
  conditioned: ["C"],
  nodes: [
    {id: "C", x: 0.5, y: 0.12, label: "C"},
    {id: "D", x: 0.18, y: 0.85, label: "D"},
    {id: "Y", x: 0.82, y: 0.85, label: "Y"}
  ],
  edges: [
    {source: "C", target: "D", status: "blocked"},
    {source: "C", target: "Y", status: "blocked"},
    {source: "D", target: "Y", status: "open-causal"}
  ]
})

How to read this figure (and every DAG on this page). C is drawn as a box because we are conditioning on it. The two arrows out of C are gray dashed — the backdoor path D \leftarrow C \rightarrow Y is blocked at the fork. The only surviving open path is the green causal arrow D \rightarrow Y. That is exactly what a good confounder control achieves: kill the red, keep the green.

2. The Indirect Confounder (Proxy for an Unobserved Confounder)

D \leftarrow U \rightarrow Y, \quad U \rightarrow W

U is an unobserved confounder, and W is a measured variable caused by U (a “proxy” or descendant of U). You cannot condition on U directly, but conditioning on W can partially block the backdoor path.

Important caveat: Conditioning on W does not fully block the path through U unless W perfectly captures all variation in U. It reduces confounding but does not eliminate it. This is called partial adjustment or imperfect proxy control.

Healthcare example: Studying the effect of a hospital’s infection control protocol (D) on patient outcomes (Y). The hospital’s organizational culture (U) is unobserved but affects both protocol adoption and patient care quality. The hospital’s staff turnover rate (W) is observable and caused by organizational culture. Conditioning on turnover rate partially blocks the confounding, though some residual bias remains.

gdag_drawDAG({
  width: Math.min(440, width), height: 200,
  conditioned: ["W"],
  nodes: [
    {id: "U", x: 0.5, y: 0.1, label: "U"},
    {id: "W", x: 0.5, y: 0.92, label: "W"},
    {id: "D", x: 0.12, y: 0.5, label: "D"},
    {id: "Y", x: 0.88, y: 0.5, label: "Y"}
  ],
  edges: [
    {source: "U", target: "D", status: "open-noncausal"},
    {source: "U", target: "Y", status: "open-noncausal"},
    {source: "U", target: "W", status: "blocked"},
    {source: "D", target: "Y", status: "open-causal"}
  ]
})

The proxy W is boxed (we condition on it), so the arrow U \rightarrow W is blocked. But the backdoor D \leftarrow U \rightarrow Y stays red: conditioning on a noisy descendant of U only partially deconfounds. Unlike the previous figure, the red path never fully closes — exactly the “imperfect proxy” caveat in prose.

3. Blocking a Longer Backdoor Path

D \leftarrow A \rightarrow B \rightarrow Y

Sometimes the confounder does not directly cause both D and Y. Instead, the backdoor path runs through multiple variables. You can block it by conditioning on any variable on the backdoor path (as long as it’s not a collider or a descendant of treatment).

Healthcare example: A study of how a structured diabetes program (D) affects HbA1c (Y). Regional healthcare budget (A) determines which vårdcentraler adopt the program. Budget also determines staffing levels (B), and better staffing independently improves HbA1c. The backdoor path is D \leftarrow A \rightarrow B \rightarrow Y. Conditioning on either A (budget) or B (staffing) blocks this path.

You don’t need to condition on the “source” confounder. Any non-collider on the backdoor path will do. This is especially useful when the true confounder is unmeasured but a variable further along the path is observed.

gdag_drawDAG({
  width: Math.min(460, width), height: 190,
  conditioned: ["B"],
  nodes: [
    {id: "A", x: 0.32, y: 0.12, label: "A"},
    {id: "B", x: 0.68, y: 0.12, label: "B"},
    {id: "D", x: 0.1, y: 0.85, label: "D"},
    {id: "Y", x: 0.9, y: 0.85, label: "Y"}
  ],
  edges: [
    {source: "A", target: "D", status: "blocked"},
    {source: "A", target: "B", status: "blocked"},
    {source: "B", target: "Y", status: "blocked"},
    {source: "D", target: "Y", status: "open-causal"}
  ]
})

The backdoor here is the chain D \leftarrow A \rightarrow B \rightarrow Y. Boxing B (staffing) breaks the chain at its middle, so all three backdoor arrows go gray dashed — even though we never touched the original source A (budget). Boxing A instead would close the same path one step earlier.

Bad Controls

4. The Mediator (Overcontrol Bias)

D \rightarrow M \rightarrow Y

M lies on the causal path from D to Y. Conditioning on M blocks part of the causal mechanism, so the estimated effect reflects only the direct effect (D \rightarrow Y), not the total effect.

Action: Do not condition on M if you want the total effect. Only condition on it if you specifically want the controlled direct effect.

Healthcare example: A Swedish region implements a new digital health platform (D) for managing chronic heart failure. The platform increases medication adherence (M), which reduces hospital readmissions (Y). If the researcher controls for medication adherence, she blocks the main causal mechanism. She would conclude the platform “doesn’t work” — when in fact it works precisely through adherence.

\text{Platform} \rightarrow \text{Adherence} \rightarrow \text{Readmissions}

Overcontrol bias is one of the most common mistakes in applied health research. When a policy or treatment works through an intermediate mechanism, controlling for that mechanism makes the treatment appear ineffective. If you find that adding a covariate to your regression makes a significant treatment effect disappear, ask yourself: could this covariate be a mediator rather than a confounder?

gdag_drawDAG({
  width: Math.min(420, width), height: 150,
  conditioned: ["M"],
  nodes: [
    {id: "D", x: 0.1, y: 0.5, label: "D"},
    {id: "M", x: 0.5, y: 0.5, label: "M"},
    {id: "Y", x: 0.9, y: 0.5, label: "Y"}
  ],
  edges: [
    {source: "D", target: "M", status: "blocked"},
    {source: "M", target: "Y", status: "blocked"}
  ]
})

This is the mirror image of a confounder. M sits on the causal path, so boxing it turns the green mechanism gray dashed — you have switched off the very channel through which the treatment works. The total effect now reads as smaller (or zero), not because of bias in the statistical sense but because you changed the estimand — the quantity you are trying to estimate — from the total effect to the direct effect.

The bar below decomposes that total effect for the heart-failure platform story. The total effect (0.50) splits into a direct channel D \rightarrow Y (0.30) and an indirect channel through the mediator D \rightarrow M \rightarrow Y (0.50 × 0.40 = 0.20). Conditioning on the mediator M erases the indirect slice, leaving only the direct slice — which is why a platform that truly works can look weak once you “adjust for” adherence.

5. The Collider (Selection Bias / Collider Bias)

D \rightarrow L \leftarrow Y

L is a common effect of D and Y. Without conditioning, D and Y are independent through this path (the collider is blocked by default). Conditioning on L opens this path and creates a spurious association.

Action: Do not condition on L or on any descendant of L.

Healthcare example: Studying whether a new cancer treatment (D) affects 5-year survival (Y). Both the treatment and the patient’s overall prognosis affect whether the patient is hospitalized (L) in year 1:

\text{Treatment} \rightarrow \text{Hospitalization} \leftarrow \text{5-year survival}

If the researcher restricts the analysis to hospitalized patients only (conditioning on L), she opens the collider. Among hospitalized patients, those who received the new treatment but were still hospitalized must have had particularly poor prognoses. This creates a spurious negative association between treatment and survival — collider bias.

Common forms of collider conditioning in healthcare research:

  • Restricting the sample: Analyzing only hospitalized patients, only survivors, only patients who completed treatment
  • Adjusting for post-treatment variables: Controlling for side effects, compliance, or intermediate test results
  • Selection into the study: If both treatment and outcome affect who ends up in your dataset (e.g., referral bias), the sample itself is conditioned on a collider
gdag_drawDAG({
  width: Math.min(420, width), height: 170,
  conditioned: ["L"],
  nodes: [
    {id: "D", x: 0.12, y: 0.18, label: "D"},
    {id: "Y", x: 0.88, y: 0.18, label: "Y"},
    {id: "L", x: 0.5, y: 0.88, label: "L"}
  ],
  edges: [
    {source: "D", target: "L", status: "open-noncausal"},
    {source: "Y", target: "L", status: "open-noncausal"},
    {source: "D", target: "Y", status: "open-causal"}
  ]
})

The danger of a collider is the opposite of a confounder. Left alone, the path D \rightarrow L \leftarrow Y is already blocked — colliders block by default. Boxing L opens it, which is why both arrows into L are drawn red: conditioning manufactures a spurious association between D and Y on top of the genuine green effect. You will watch this spurious association appear from scratch in the “Explaining-Away” demo further down the page.

6. The Descendant of a Collider

D \rightarrow L \leftarrow Y, \quad L \rightarrow S

S is a descendant of the collider L. Conditioning on S partially opens the collider path, because knowing S gives information about L.

Healthcare example: In the cancer treatment example above, suppose hospitalization (L) leads to a specific billing code (S). Conditioning on the billing code partially opens the collider path, introducing a weaker but still present version of collider bias.

Action: Do not condition on descendants of colliders either. Any variable that is caused (directly or indirectly) by a collider should be treated with suspicion.

function mulberry32(a) {
  return function() {
    a |= 0; a = a + 0x6D2B79F5 | 0;
    var t = Math.imul(a ^ a >>> 15, 1 | a);
    t = t + Math.imul(t ^ t >>> 7, 61 | t) ^ t;
    return ((t ^ t >>> 14) >>> 0) / 4294967296;
  }
}

  let u2 = rng();
  return Math.sqrt(-2 * Math.log(u1)) * Math.cos(2 * Math.PI * u2);

controlRegression = {
  const condC = controls.includes("C — Confounder (severity)");
  const condM = controls.includes("M — Mediator (biomarker)");
  const condL = controls.includes("L — Collider (hospitalization)");

  const d = controlData;

  // Residualize against the FULL control set (proper multiple-regression
  // partialling, via Frisch–Waugh). The controls are first Gram–Schmidt
  // orthogonalized so that correlated controls (e.g. C and M, which are both
  // downstream of the same variables) are partialled out jointly, not one at a
  // time. A naive one-control-at-a-time residualization equals OLS only when
  // the controls are mutually orthogonal; here they are not, so it would bias
  // the multi-control estimates (e.g. the C+M controlled direct effect).
  function residualize(outcome, ctrls) {
    if (ctrls.length === 0) return outcome.slice();
    // Build an orthogonal (centered) basis spanning the controls.
    const basis = [];
    for (const raw of ctrls) {
      const mv = d3.mean(raw);
      let v = raw.map(x => x - mv);
      for (const b of basis) {
        const denom = d3.sum(b, x => x * x);
        if (denom === 0) continue;
        const coef = d3.sum(b.map((x, i) => x * v[i])) / denom;
        v = v.map((x, i) => x - coef * b[i]);
      }
      basis.push(v);
    }
    // Project the outcome onto the orthogonal basis and subtract.
    let resid = outcome.slice();
    for (const b of basis) {
      const denom = d3.sum(b, x => x * x);
      if (denom === 0) continue;
      const coef = d3.sum(b.map((x, i) => x * resid[i])) / denom;
      resid = resid.map((x, i) => x - coef * b[i]);
    }
    return resid;
  }

  const ctrlArrays = [];
  if (condC) ctrlArrays.push(d.map(v => v.C));
  if (condM) ctrlArrays.push(d.map(v => v.M));
  if (condL) ctrlArrays.push(d.map(v => v.L));

  const residD = residualize(d.map(v => v.D), ctrlArrays);
  const residY = residualize(d.map(v => v.Y), ctrlArrays);

  const mrD = d3.mean(residD);
  const mrY = d3.mean(residY);
  const cov = d3.sum(residD.map((rd, i) => (rd - mrD) * (residY[i] - mrY)));
  const varD = d3.sum(residD.map(rd => (rd - mrD) ** 2));
  const beta = cov / varD;

  // Determine what this estimate represents
  let interpretation = "";
  let biasDirection = "";

  if (!condC && !condM && !condL) {
    interpretation = "Confounded estimate (backdoor open through C)";
    biasDirection = "upward";
  } else if (condC && !condM && !condL) {
    interpretation = "✓ Total causal effect (backdoor blocked, causal path intact)";
    biasDirection = "none";
  } else if (!condC && condM && !condL) {
    interpretation = "Neither total nor direct effect (backdoor open + mediator blocked)";
    biasDirection = "mixed";
  } else if (!condC && !condM && condL) {
    interpretation = "Confounded + collider bias (both bias sources active)";
    biasDirection = "unpredictable";
  } else if (condC && condM && !condL) {
    interpretation = "Controlled direct effect (backdoor blocked, mediator blocked)";
    biasDirection = "none";
  } else if (condC && !condM && condL) {
    interpretation = "Total effect + collider bias introduced";
    biasDirection = "collider";
  } else if (!condC && condM && condL) {
    interpretation = "Confounded + mediator blocked + collider bias";
    biasDirection = "multiple";
  } else {
    interpretation = "Direct effect + collider bias introduced";
    biasDirection = "collider";
  }

  return { beta, interpretation, biasDirection };
}

How to Read the Control Laboratory

The laboratory has three linked pieces: a live DAG at the top, a single-sample estimate in the middle, and a sampling-distribution strip at the bottom. Here is the guided tour:

Step 1: Watch the DAG recolor. Toggle the checkboxes and look at the DAG first, before the numbers. When you box C, the two red backdoor arrows turn gray (blocked). When you box M, the green mediator arrows turn gray (mechanism switched off). When you box L, the two collider arrows flip from gray to red (a spurious path opens). The colors are the same legend used by all six pattern figures above.

Step 2: Read the single-sample estimate. The “Your Estimate” card is computed on one fixed dataset (seed-locked) so the page’s worked numbers in What to Try never drift. Every checked control enters a single multiple regression jointly (not one at a time), so combinations like C + M behave exactly like a real fitted model. Compare it to the true total (green) and direct (ochre) values.

Step 3: Separate bias from noise. Click Draw new sample (×200). The strip resimulates the DAG 200 times and plots where the estimate lands each time. Two distinct things to read:

  • The center of the cloud (dark “Mean estimate” line) is the bias. If it sits on the green “true total” line, the control set is unbiased for the total effect; if it sits away from it, you have systematic bias.
  • The width of the cloud (and the “Sampling SD” card) is sampling noise — the spread you would see even with a perfectly unbiased estimator, just from finite data.

Step 4: Diagnose each control set. With nothing controlled, the cloud centers well above the green line (confounding bias). With C alone, the cloud snaps onto the green line — center on target, narrow spread. With L added, the center drifts off again (collider bias) — even though the spread barely changes. This is the whole point: more controls did not make the estimate noisier, they made it wrong. Bias and noise are different failures, and only the strip lets you see both at once.

What to Try

Work through each combination systematically:

  1. Nothing controlled: The estimate (~1.0) is inflated by confounding through C — roughly double the true total effect of 0.50.

  2. Only C (Confounder): The estimate drops to ~0.50 — the true total causal effect. The backdoor is blocked and the causal path is intact. ✓

  3. Only M (Mediator): The estimate drops, but the backdoor is still open. You get something uninterpretable — neither the total effect nor the direct effect.

  4. Only L (Collider): The estimate changes unpredictably. You’ve opened a collider path and the backdoor is still open. Two sources of bias.

  5. C + M (Confounder + Mediator): The estimate drops to ≈ 0.30 — the controlled direct effect. The backdoor is blocked and the mediator path is blocked, so this targets the direct effect D \rightarrow Y rather than the total effect — a different causal question, not a biased version of the same one. (Draw a new sample a few times: the cloud centers right on the ochre “true direct” line, with only sampling noise around it.)

  6. C + L (Confounder + Collider): The estimate deviates from 0.50. The backdoor is blocked (good), but the collider is opened (bad). A new source of bias is introduced.

The only combination that recovers the total causal effect is conditioning on C alone. Adding any other variable either changes what you’re estimating (M) or introduces new bias (L). More controls ≠ less bias.

  for (const p of pts) p.selected = p.sum > thr;
  return pts;
}

// OLS slope of Y on D over a given point set.
gdag_slope = function(pts) {
  if (pts.length < 2) return 0;
  const mx = d3.mean(pts, p => p.D);
  const my = d3.mean(pts, p => p.Y);
  const cov = d3.sum(pts, p => (p.D - mx) * (p.Y - my));
  const vx = d3.sum(pts, p => (p.D - mx) ** 2);
  return vx === 0 ? 0 : cov / vx;
}

  const line = xs.map(x => ({ D: x, Y: my + b * (x - mx) }));

How to Read the Explaining-Away Demo

Step 1: Start with the toggle off. Every point is plotted and the fitted line is essentially flat (slope near 0). This is the truth: D and Y were generated independently. Knowing a patient’s treatment D tells you nothing about their outcome Y.

Step 2: Look at the slate points. The darker slate points are the “selected” subsample — patients for whom D + Y cleared the admission threshold. They form a triangular cloud cut off along a diagonal line D + Y = \text{constant}. That diagonal boundary is the collider doing its work.

Step 3: Flip the toggle on. Now we condition on L by keeping only the selected (slate) points, and the fitted line tilts downward. Among the selected, a patient with a high D must have a low Y to have made the cut — and vice versa. High D “explains away” the need for high Y. A negative correlation appears out of nothing.

Step 4: Re-draw to confirm it is not a fluke. Click Draw new sample a few times. The flat line stays flat; the conditioned line stays tilted. The spurious negative slope is structural, not sampling noise — it returns on every draw. This is exactly the cancer-survival trap from pattern 5: restrict to hospitalized patients and an effective treatment can look harmful.

See It: M-Bias and the Pre-Treatment Collider “Butterfly”

Here is the pattern that breaks the comforting rule “just control for anything measured before treatment.” Consider a variable Z that is measured at baseline — genuinely pre-treatment — yet is a collider between two unobserved causes. Its DAG is the famous butterfly (or M-structure):

D \leftarrow U_1 \rightarrow Z \leftarrow U_2 \rightarrow Y

Both U_1 and U_2 are unobserved. Z is a pre-treatment collider sitting between them. Leave Z alone and there is no open path from D to Y — the collider blocks it. Control for Z “to be safe” and you open a biasing path that did not exist before.

// correlation appear only when Z is residualized out.
gdag_mbiasFit = {
  const n = 4000;
  const rng = mulberry32(20240619);  // seed-locked: stable cards across renders
  const D = [], Y = [], Z = [];
  for (let i = 0; i < n; i++) {
    const U1 = randn(rng), U2 = randn(rng);
    Z.push(U1 + U2 + randn(rng) * 0.5);   // pre-treatment collider of U1, U2
    D.push(U1 + randn(rng));               // D driven by U1 only (NOT by Y)
    Y.push(U2 + randn(rng));               // Y driven by U2 only (NOT by D) → true D→Y = 0
  }
  // Slope of Y on D, optionally residualizing out Z.
  function resid(v, ctrl) {
    const mc = d3.mean(ctrl), vc = d3.sum(ctrl, c => (c - mc) ** 2);
    const cov = d3.sum(ctrl.map((c, i) => (c - mc) * v[i]));
    const b = cov / vc;
    return v.map((x, i) => x - b * (ctrl[i] - mc));
  }
  function slope(x, y) {
    const mx = d3.mean(x), my = d3.mean(y);
    const cov = d3.sum(x.map((xi, i) => (xi - mx) * (y[i] - my)));
    const vx = d3.sum(x.map(xi => (xi - mx) ** 2));
    return cov / vx;
  }

  const rD = resid(D, Z), rY = resid(Y, Z);
  const controlled = slope(rD, rY);

  // Z-residualized D vs Y (what the regression actually sees once Z is added).

How to Read the Butterfly Demo

Step 1: Note what is unobserved. U_1 and U_2 are confounders you never measure. The only variable you can put in a regression here is Z — and it is measured at baseline, before treatment. Every instinct says it is a safe control.

Step 2: Leave Z alone (toggle off). All arrows are gray: the path D \leftarrow U_1 \rightarrow Z \leftarrow U_2 \rightarrow Y is blocked at the collider Z. The numeric cards confirm both estimates sit at 0.00, matching the true zero effect. Doing nothing is correct.

Step 3: Control for Z (toggle on). The four arrows light up red: boxing the collider Z opens U_1 \rightarrow Z \leftarrow U_2, which links U_1 and U_2, and through them D and Y. The “Z controlled” card jumps away from zero — a bias you created by adding a controllably “clean” baseline covariate.

Step 4: Take the lesson. Pre-treatment timing is not a license to control. M-bias is the reason “adjust for everything you measured before treatment” is wrong as a blanket rule. You still need the DAG to tell a pre-treatment confounder (control it) from a pre-treatment collider (leave it alone).

When faced with a potential control variable, ask these questions in order:

Step 1: Is the variable a descendant of treatment (D)?

  • If yes → it is either a mediator or a collider (or both). Do not condition on it for the total effect. Stop here.
  • If no → proceed to Step 2.

Step 2: Does the variable lie on a backdoor path from D to Y?

  • If yes → conditioning on it blocks that backdoor path. This is a good control. Include it.
  • If no → proceed to Step 3.

Step 3: Is the variable a collider on any path from D to Y?

  • If yes → conditioning on it opens a spurious path. This is a bad control. Exclude it.
  • If no → the variable is not on any path between D and Y. Conditioning on it neither helps nor hurts identification, but may improve precision if it predicts Y.

Collider descendant L \rightarrow S No Partially opens the collider path
Pre-treatment collider (M-bias) D \leftarrow U_1 \rightarrow Z \leftarrow U_2 \rightarrow Y No Opens a path even though Z precedes D
Precision variable W \rightarrow Y only Optional Improves precision, doesn’t affect bias

The “Table 2 Fallacy”

A widespread practice in medical research is to report a regression table with the treatment effect and the coefficients on all control variables, interpreting each coefficient as a causal effect. This is known as the Table 2 fallacy (Westreich and Greenland, 2013).

The problem: the adjustment set that identifies the causal effect of D on Y is not the same set that identifies the causal effect of C on Y. Each causal question requires its own DAG analysis and its own adjustment set. The coefficients on control variables in a single regression have no causal interpretation unless each variable’s confounders are also properly controlled.

Healthcare example: A study estimates the effect of a diabetes drug (D) on HbA1c (Y), controlling for BMI (C), exercise (M), and age (A). The coefficient on D may be a valid causal estimate (if \{C, A\} satisfies the backdoor criterion). But the coefficient on BMI from the same regression is not a valid causal estimate of BMI’s effect on HbA1c, because the confounders of BMI-HbA1c may differ entirely from the confounders of D-Y.

Common Traps in Healthcare Research

Trap 1: Controlling for Intermediate Outcomes

A hospital evaluates whether a new care pathway (D) reduces 30-day mortality (Y). The pathway works by reducing time-to-treatment (M). A researcher controls for time-to-treatment, finds “no effect” of the pathway, and concludes it’s useless.

Diagnosis: M is a mediator. Conditioning on it blocks the mechanism. The pathway does work — through M.

Trap 2: Restricting to Survivors (Survivorship Bias)

A study of an intensive care intervention (D) on long-term quality of life (Y) restricts the sample to patients who survived 90 days. Both the intervention and pre-existing health affect 90-day survival (L).

Diagnosis: 90-day survival is a collider (D \rightarrow L \leftarrow \text{Pre-existing health} \rightarrow Y). Conditioning on it opens a spurious path. Among survivors, treated patients with poor outcomes must have had unusually good pre-existing health.

Trap 3: Controlling for a Variable Affected by Treatment

A trial of a dietary intervention (D) measures both weight loss (M) at 6 months and cardiovascular events (Y) at 2 years. The researcher controls for 6-month weight loss. Depending on the DAG, this could block a mediator or open a collider if weight loss is also affected by unmeasured variables that affect Y.

Diagnosis: Always ask “was this variable measured after treatment was assigned?” If yes, it may be a descendant of D. Draw the DAG before deciding.

Math Details: The do-Calculus and Beyond the Backdoor

When the Backdoor Criterion Is Not Enough

The backdoor criterion is powerful but not universal. Some causal effects can be identified even when no valid backdoor adjustment set exists. Pearl’s do-calculus (1995) provides three general rules for manipulating interventional distributions:

Notation at a glance (the symbols in the three rules below):

  • do(D) — the intervention operator: “set D by force,” as in a randomized trial, rather than merely observing the D that nature handed out. P(Y \mid do(D)) is the distribution of Y if we reached in and assigned D.
  • \perp — (conditional) independence: Y \perp Z \mid D, W reads “once we know D and W, learning Z tells us nothing more about Y.” It is the algebraic side of d-separation.
  • \mathcal{G}_{\overline{D}} — the DAG with all arrows into D deleted (the overline = “incoming arrows cut”). \mathcal{G}_{\underline{Z}} deletes arrows out of Z (underline = “outgoing arrows cut”). These “graph surgeries” let you check the d-separation condition on a temporarily edited graph.

Rule 1 (Insertion/deletion of observations): P(Y \mid do(D), Z, W) = P(Y \mid do(D), W) \quad \text{if } Y \perp_{\mathcal{G}_{\overline{D}}} Z \mid D, W

How to read this: “Observing Z doesn’t change Y’s distribution after intervening on D and conditioning on W, provided that Y and Z are d-separated in the modified graph where arrows into D are removed.”

Rule 2 (Action/observation exchange): P(Y \mid do(D), do(Z), W) = P(Y \mid do(D), Z, W) \quad \text{if } Y \perp_{\mathcal{G}_{\overline{D}\underline{Z}}} Z \mid D, W

How to read this: “Intervening on Z is the same as observing Z” under certain d-separation conditions in a modified graph.

Rule 3 (Insertion/deletion of actions): P(Y \mid do(D), do(Z), W) = P(Y \mid do(D), W) \quad \text{if } Y \perp_{\mathcal{G}_{\overline{D}\overline{Z(S)}}} Z \mid D, W

where S is the set of nodes in Z that are not ancestors of any W-node in \mathcal{G}_{\overline{D}}.

These rules are complete: if a causal effect can be identified from observational data given the DAG, the do-calculus can derive the identifying formula. Algorithms like the ID algorithm (Tian and Pearl, 2002) automate this process.

The Front-Door Criterion

One important special case that goes beyond the backdoor: when an unmeasured confounder exists but the treatment’s entire effect flows through a measurable mediator:

D \leftarrow U \rightarrow Y, \quad D \rightarrow M \rightarrow Y

No backdoor adjustment works (because U is unmeasured). But the front-door criterion identifies the effect through:

P(Y \mid do(D)) = \sum_m P(M = m \mid D) \sum_d P(Y \mid M = m, D = d) P(D = d)

The front-door criterion is covered in more detail in later modules. For now, the key point is: the backdoor criterion is the first tool to try, but not the last. The DAG can reveal identification strategies you might not have considered.

Exercise 1

For each of the following variables in a study of how a new surgical technique (D) affects post-operative infection rates (Y) in Swedish hospitals, classify it as a confounder, mediator, collider, or irrelevant, and state whether you should condition on it:

  1. Surgeon’s years of experience
  2. Surgical wound healing speed (measured post-operatively)
  3. Whether the patient was transferred to the ICU (caused by both the technique used and infection)
  4. The patient’s blood type
Show answer

(a) Surgeon’s years of experience → Confounder. More experienced surgeons may both adopt the new technique more readily and have better outcomes independently. This creates a fork: D \leftarrow \text{Experience} \rightarrow Y. Condition on it ✓.

(b) Surgical wound healing speed → Mediator. The new technique may cause faster wound healing, which in turn reduces infections: D \rightarrow \text{Healing speed} \rightarrow Y. Conditioning on it blocks the causal mechanism and gives you the direct effect only. Do not condition on it for the total effect ✗.

(c) ICU transfer → Collider. Both the technique and the infection outcome can affect whether the patient goes to the ICU: D \rightarrow \text{ICU} \leftarrow Y. Conditioning on it (or restricting to ICU patients) opens a spurious path. Do not condition on it ✗.

(d) Blood type → Irrelevant (precision variable). Blood type plausibly does not affect the choice of surgical technique and has at most a minor effect on infection risk. If it affects Y but not D, it can be included for precision but is not required for identification. Optional — including it may slightly improve efficiency.

Exercise 2

Using the interactive above, find the combination of controls that estimates the controlled direct effect (the effect of D on Y not through M). What value should the estimate be close to? Why is this a different quantity from the total effect?

Show answer

To estimate the controlled direct effect, condition on C (Confounder) and M (Mediator):

  • Conditioning on C blocks the backdoor path D \leftarrow C \rightarrow Y → removes confounding.
  • Conditioning on M blocks the indirect causal path D \rightarrow M \rightarrow Y → removes the mediated effect.

What remains is only the direct path D \rightarrow Y, which has a coefficient of 0.30 in the data-generating process. So the estimate should be close to 0.30.

This differs from the total effect (0.50) because the total effect includes both the direct and indirect pathways:

  • Direct: D \rightarrow Y = 0.30
  • Indirect: D \rightarrow M \rightarrow Y = 0.50 \times 0.40 = 0.20
  • Total: 0.30 + 0.20 = 0.50

The controlled direct effect answers: “What would happen if the treatment changed but the mediator was held fixed?” The total effect answers: “What would happen if the treatment changed, allowing all mechanisms to operate?” These are different causal questions with different policy implications. A platform that improves outcomes only through adherence (M) has a total effect but zero direct effect.

A Swedish epidemiologist studies whether air pollution (D) increases asthma hospitalizations (Y) using registry data. She has the following variables available:

  • Pollen count (affects respiratory health independently)
  • Medication use (affected by pollution exposure and asthma severity)
  • Number of ER visits in the last month (affected by both medication use and asthma severity)

Draw a plausible DAG for this setting. Then identify which variables are good controls, which are bad controls, and which are optional.

Show answer

Plausible DAG:

  • Temperature → D (pollution), Temperature → Y (hospitalizations)
  • Pollen → Y
  • D → Medication use, Y (asthma severity) → Medication use
  • Medication use → ER visits, Y (asthma severity) → ER visits

Classification:

  • Temperature → Confounder (fork: D \leftarrow \text{Temp} \rightarrow Y). Good control — condition on it ✓.

  • Pollen count → Precision variable (\text{Pollen} \rightarrow Y only, no arrow to D). Optional — including it does not reduce confounding but may improve statistical precision.

Exercise 4

Explain the Table 2 fallacy in your own words. Then give a concrete example where a single regression produces a valid causal estimate for one coefficient but not for another, even though both variables appear in the same model.

Show answer

The Table 2 fallacy occurs when researchers interpret all coefficients in a regression as causal effects, when the regression was designed to identify the causal effect of one specific variable. The adjustment set that blocks all backdoor paths for the treatment of interest does not automatically block all backdoor paths for the control variables.

Concrete example: A researcher estimates the effect of a new rehabilitation program (D) on physical function (Y), controlling for patient age (A) and pre-existing diabetes status (C).

  • DAG: A \rightarrow D, A \rightarrow Y, C \rightarrow Y, and also an unobserved variable U such that U \rightarrow C and U \rightarrow Y.

  • For the treatment effect (D \rightarrow Y): The adjustment set \{A, C\} satisfies the backdoor criterion. The coefficient on D is a valid causal estimate ✓.

  • For the age effect (A \rightarrow Y): The backdoor criterion for A requires blocking any backdoor paths from A to Y. If there are unobserved common causes of A and Y (e.g., birth cohort effects, genetic factors) that are not in the model, the coefficient on A is confounded. Invalid ✗.

  • For the diabetes effect (C \rightarrow Y): The unobserved confounder U creates a backdoor C \leftarrow U \rightarrow Y. Since U is not in the model, the coefficient on C is confounded. Invalid ✗.

Lesson: Each causal question requires its own DAG analysis. You cannot “read off” causal effects for multiple variables from a single regression.

Key takeaways

  • A control is “good” only if it blocks a non-causal path without opening another or closing a causal one — the causal role, not the correlation with D and Y, decides.
  • Conditioning on a mediator causes overcontrol bias: it switches off the mechanism and changes the estimand from the total effect to the direct effect.
  • Conditioning on a collider — or its descendant, or by sample restriction (only survivors, only hospitalized) — manufactures a spurious association out of nothing.
  • “Pre-treatment” is no guarantee of safety: in the M-bias butterfly D \leftarrow U_1 \rightarrow Z \leftarrow U_2 \rightarrow Y, controlling for the baseline collider Z opens a biasing path.
  • More controls do not mean less bias — adding the wrong variable shifts the estimate’s center (bias) without shrinking its spread — and the Table 2 fallacy reminds you each coefficient needs its own DAG.
